CSS - Positioning

When it comes to positioning you should think of it as the bones or supporting beams of your website. If you don't set them up correctly it will ultimately make your website appear dull and messy. Early website design used tables/boxes in order to create the structure of a website - this is now an outdated practice which is not recommended due to it's lack of flexibility. While some visual features and interactivity must be done through other code such as JavaScript, the bulk of appearance comes from CSS.

Arguably the most important thing to learn for appearance and positioning is 'The Box Model', This represents the structure of almost all elements and the impact which the content, padding, border and margin have upon the appearance of said item and the webpage. This is represented in the accompanying image. Also of great importance are the tags 'display' and 'position', the former determines the behaviour of an element and how it will place itself on the webpage while the latter allows you to manipulate this position in a more targeted manner. The two tags are commonly used together to create the desired effect.

CSS - Referencing Color

Color is vital when it comes to any web-design and not just vibrant color but also your 'non-colors', the grayscale from white to black is just as important to give a feeling of immersion. Whether right to do so or not, people trust what (or who) they find attractive. For example, if you have two almost identical products being marketed - people will consistenly purchase from the company that has the more visually engaging website as it instills a sense of confidence in the company - i.e. "This website is impressive and suits my taste/style, they know what they're doing - they probably know what they're doing with x product as well". It may sound ridiculous but it's true.

Color is applied to all elements of a webpage but we are more specifically talking about it's assignment to borders (color:), text (color:) and backgrounds (background-color:). Color can be presented and applied in four different styles:

  • Named Value

    • Named value refernecing is the simplest and easiest way to color your webpage in that you simply set the value to a named color such as 'red' and that's that! While there is an extensive default library of colors (approximately 147 with 17 agreed standard colors: aqua, black, blue, fuchsia, gray, grey, green, lime, maroon, navy, olive, purple, red, silver, teal, white and yellow), this is still extremely limiting in the scope of what colors can be used using other methods such as Hex (as you will see shortly). Using named colors is generally considered bad practice and should be limited to one-off's and 'pre-formatting' stages of projects (i.e. where you are building content and have yet to decide a theme). The syntax for a named value would be:
    div {
    background-color: teal;
    }

  • Hexadecimal (Hex)

    • Hexadecimal or Hex is the most commonly used color referencing format and appears as a hashtag followed by either 3 or 6 numbers/letters. At first glance, Hex can be a little intimadating as it is a syntax and representation that people are not commonly familiar with. The digits that follow the hashtag are pairings that represent a mix or ratio of 'red', 'blue' and 'green'. These digits range from '00' to 'FF' which represents a scale from 0 to 256 - for clarity you can view this here. For example, take the color 'teal', this color would be represented as '#008080' where [#]=hastag, [00]=red, [80]=green & [80]=blue. In terms of syntax this would look as follows:
    div {
    background-color: #008080;
    }

  • Red-Green-Blue (RGB)

    • Red-Green-Blue or RGB is the second most commonly used color referencing format and is popular due to it's range of color selection and accessibility for users. In theory it follows the same syntax as as Hex in that it presents a combination of the values Red, Green and Blue. Each value is presented numerically on a scale from 0 to 256. The syntax is presented as follows:
    div {
    background-color: rgb(0, 129, 129);
    }

  • Hue-Saturation-Lightness(HSL)

    • Hue-Saturation-Lightness or HSL is another color referencing format. It is one of the least commonly used syntax, largely because of it's lack of accessibility. Those who are familiar with phtoography or digital design will understand the influence of hue, saturation and lightness on an image or photo. As with previous references, HSL uses a scale for three identifiers as a default however it can also be present as HSLa where a represents opacity. The scale for Hue is presented from 0 to 360 (which represents a degree on a color-wheel for which color it represents), Saturation and Lightness are presented as a percentile between 0-100% and a, or opacity on a scale between 0 and 1. Where HSLa becomes useful is when you have colors which are not static and need to change in response to script, the division of these elements allows one to be adjsuted in isolation to the others. The syntax for HSLa is presented as follows:
    div {
    background-color: hsla(120,240,61,1);
    }

  • Bringing It All Together

    • All of the above, represent different ways in which you can reference color in css. Take a look at the below, each of these syntax represent the color 'lavender':

    div { background-color: lavender }

    div { background-color: #e6e6fa }

    div { background-color: rgb(230, 230, 250) }

    div { background-color: hsla(240, 67%, 94%, 1) }

CSS - Referencing Typography

When it comes to the appearance of a website, it is not just the positioning and color of a section that makes it appealing but also the appearance of the content within these sections. One of the ways that this can be achieved is through font-styling. While the default for all web browsers is 'Times New Roman', other fonts can help complete the styling or theme that the designer is going for. When applying font-styles to a webpage it is important to not go over the top, three font-styles should be limited to each webpage and it is recommended to avoid cursive - ignoring either of these rules will likely result in your website looking messy and hard to read. Not all fonts are made equal however, different browsers support different font sets with 'Edge' and 'Chrome' supporting the largest number of local fonts. This presents a challenge when creating content as you will need to ensure thast your webpage is visible on multiple browsers. CSS presents a syntax that helps to combat this, when typing preferred text you chan select 'fall-back fonts', whereby if the first font is available, then the next will be selected and so on. This is particularly important when referencing external fonts which while unlikely, are more prone to fail that local fonts. The syntax for this looks as follows:
p {
font-family: helvetica, arial, sans serif
}
In this example, the p element will use the font helvetica, if this font is not available then it will use arial and finally if this is not available it will use the primary local sans serif style font.

Many choose to rely upon local font sets, but this can be limiting and prevent the designer for delivering the intended web-design. The alternative is to link your document to external css files, this means that when the user opens the webpage it calls a datafile with the corresponding font specifications and allows for this to be viewable. There are a number of websites that offer a library of font-styles which can be used in your webpage. While this applies to a styling element, the link is applied to the top of the html document. The syntax for this is as follows:
<link href="https://fonts.googleapis.com/css?family=Lato" rel="stylesheet">
In this example users are now able to reference and call upon the 'Lato' font and apply this throughout their webpage and be referenced in the css file.

If the designer wishes for a font to be called upon within the stylesheet then this can be done using @font-face property. This is more common when a developer is creating a custom stylesheet that they wish to apply to multiple pages or projects - a common practice. In order to do this, a user will apply the below syntax to the top of their css document:@font-face {
font-family: 'Glegoo';
src: url('../fonts/Glegoo-Regular.ttf') format('truetype');
}

CSS - Typography Themes

Below you can find a range of font sets, some of which are web-safe fonts and some of which are imported custom fonts (or embedded fonts). When selecting a font for your website/prohect, it is important that you consider a number of factors.

First and foremost is readability - it may seem enticing to use a cursive font which has an attractive flow to it but this is not a smart choice for the body of a webpage as this is usually hard to read. For large bodies of text it is good practice to choose a font-style which is 'serif', this means that the letters have 'feet' or extensions - these fonts typically have letters which are closer together and flow well, making them prime for long passages of text.

Secondly, you should consider which fonts pair well together, and which contrast one another effectively to draw distinction from a heading, abstract, quote or body of text. For example, you may consider a sans serif font for your headings and a serif font for the body of text.

Thirdly, experiment - just because you have chosen a font at the onset of release does not mean that the font needs to stay the same - it has been found that different fonts instill different reader/customer reactions. Some fonts (and colors) have led to more sales than others because of how they draw the reader in and give a sense of confidence or comfort.

Finally, consider the tone of the page - serious work calls for serious font - not comic sans! Trust your gut, if it is visually appealing to you and you think it fits, run with it.
